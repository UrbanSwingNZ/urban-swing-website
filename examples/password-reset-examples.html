This page demonstrates all the different ways to use the password reset utilities in the Urban Swing website.

Example 1: Simple Modal (Recommended)

The easiest implementation. Just call showPasswordResetModal() when a button is clicked.

Use Case: Perfect for "Forgot Password?" links on login pages.

Example 2: Modal with Pre-filled Email

Pre-fill the email field using an existing input value.

Use Case: When user has already entered their email in a login form.

Example 3: Modal with Success Callback

Execute custom code after the password reset email is sent.

Use Case: When you need to track analytics, show notifications, or redirect after success.

Example 4: Inline Form Handler

Add password reset to an existing form without showing a modal.

Use Case: When you want inline feedback without a modal popup.

Example 5: Direct API Call

Call the core function directly for complete control.

Use Case: When you need full programmatic control or custom UI.

Example 6: Custom Modal Configuration

Customize the modal title, description, and other properties.

Use Case: When you need different messaging for different contexts (student vs admin, etc.).
